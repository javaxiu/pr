# 编排 - 变量参数

# 场景

如下图这种情况下，「outer」 这个区域的文字，有时候我需要它是个常量（固定的字符串），有时候希望是个变量（动态从后台拉取的），甚至希望很灵活的交由用户决定是绑定变量还是手填固定值，这种情况下，Setter 需要怎么配置参数才能满足这种需求呢？

# 实现方式

# 数据结构的实现

设置为常量的情况

{
  componentName: "card",
  props: {
    title: "标题",
    ...otherProps,
  }
}

设置为变量的情况

{
  componentName: "card",
  props: {
    title: {
      type: "JSExpression",
      // expression 里的 this 代表 page 的实例
      value: "this.state.text",
    },
    ...otherProps,
  }
}

基于这个数据结构，如果用户直接在输入框里输入了文字，则将此节点的数据设置为第一种，如果用户绑定了变量，则使用第二种方式。

# Page 的实现

业务场景下，可能有不同的数据上下文，比如一张活动页面，上下文状态���括：用户登录信息、活动信息，这些数据可以类比 React 里的 state，需要有个地方编写和存储，因此催生这样一个特殊的页面组件：Page，它不负责具体的 UI 实现，负责管理页面的全局数据和方法，提供给不同的组件使用，其他 UI 组件都作为子组件被拖入到 Page 组件下，同时如果有变量绑定、方法绑定的需要，则可以在 JSExpression 中以 this.state 的方式使用 Page 的响应式数据；this.xx 的方式使用 Page 的方法

# Setter 的实现

很简答的一个 StringSetter + 变量绑定 setter。变量绑定的 Setter 里，可以是面向程序员的像 lowcode-engine 这种 Coding 的编辑器，可以像下拉框一样更好理解的选择绑定

# 数据响应问题

回顾 RenderEngine 里的逻辑和递归渲染的逻辑，自上而下其实是一次性解析的过程，在 node.props 处会做一次类似 transformExpression(node.props) 概念 - 可视化编辑器 (opens new window) 然而当 Page.state 发生变化时，需要如何通知到每个组件「你的 Props 变了」这件事呢，而且如何做到最小监听程度，避免每次 Page.state 上一些与当前组件无关紧要的数据变化了也马上通知到自己。这里需要引入一层 NodeWrap

const renderNode = (schema: Node) => schema ? (
   React.createElement(
     getNodeWrap(node.componentName),
     node.props,
     node.children?.map(child => renderNode(node.child))
   ) 
) : null

const getNodeWrap = (name) => {
  // HOC 组件
  const NodeWrap = (props) => {
    const [finalProps, dependencies] = transformProps(props);
    const [renderProps, setRenderProps] = useState(finalProps);
    // 这里的 dependencies 就是此组件实际依赖了哪些数据
    // ['state.title', 'onCardClick']
    // 判断如果有包含 state 的依赖，则绑定上变量监听
    // 监听 Page.state.componentDidUpdate, 得到 state 是否变化
    onPageStateChange(() => {
      if (newState.title !== oldState.title) {
        setRenderProps();
      }
    })
    return React.createElement(propsSchema.componentName, finalProps),
  }
  NodeWrap.displayName = name;
  return NodeWrap;
}

了解 Vue 或者 react-mobx 的开发者应该会很好理解这里，实际上就是 「依赖收集 - autorun」 的过程
