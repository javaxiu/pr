# 职业 - RPG式的沟通

《三体》在描述三体人沟通方式那段，印象比较深刻。三体人天然没有语言沟通，天然靠脑电波进行沟通，因此他们没有“欺骗”、“秘密”这些人类的专利，即便军事冲突，也是开打之前就知道彼此战力，直接以战力决定胜负 -- 直接不打了。基于这个沟通模式，他们个体之间的知识经验交流、传承极其高效，科技生产力因而获得很大提升。

每次遇到一些沟通上的问题的时候，就时常会想起这段故事，分享一下今年遇到的三件跟沟通有关的小事：

**一。**上半年因为负责的业务成倍增加，自己一个人忙不过来，我就把一部分早先做好的程序设计，给一个新入职的师弟讲了，从背景到面临的问题，远到未来期望的产品形态，近到眼下需要的东西，我自认为各细节都已经说得比较清楚了，然后基于这些业务需要推导出来我们接下来的工作和技术方案，师弟也连连点头称懂，因为他社会从业经验比我长，我也比较放心的交给他了。后面几天我偶尔会粗略一下他的代码，从大体框架上看起来也好像是按照预期方向实施的。但万万没想到最终做出来的效果和我预期的完全不是同一个东西，这件事让我特别苦恼。以后总不能每天像老师批改作业一样每天盯着吧？这给师弟的压力会很大，自己也没这么多时间。 **二。**最近休假了一个多星期，回来之后跟另一个师弟确认他近期的工作计划，发现从他嘴里说出来要做的事情，非常蹩脚，心想这个东西做出来有什么用呢，更诧异的是他告诉我这是我们主管想要的东西。然后借着晚饭后散步的时间，我当做闲���，对这个业务问了他几个问题。第一个是，你觉得这个程序，最终你是希望用户怎么使用的呢？或者如果你是用户，你希望你拿到的这个东西是什么样的形态？第二个问题是，为了达到你说的这个效果，如果抛开主管要求的实现方式，你打算自己如何实施？第三个问题是，主管当时跟你说这个做法的时候，你对他的提案有没有疑问？简单的三个问题，聊了半个小时，中间有一些辅助他尝试主动思考的过程，聊完之后他得出的结论和我想的基本接近了。然后我就找时间跟主管当面沟通了，发现主管想要的东西其实就是我预期的。只是因为他们沟通可能稍微有点跨服聊天了。 **三。**前天在周会上，我的小组3个师弟在周会上给大组其他同事介绍我们的产品，三个人的分享方式各有不同。我观察了一下他们的会场效果：第一个95后小伙子喜欢说段子，抛流行梗，大家会在出梗的时候尝试去看看他上下文说了什么，然后笑过之后接着埋头做自己的事情；第二个95后海归小伙讲的材料是按我教他的方式在做一些产品和技术角度的产品发展，还算有条理，但是他比较害羞，对自己说的事情不够自信，还时不时尬笑一下，缓解自己的紧张，会场效果是只有一两个人在听；第三个讲得有点国企风，就是对人和产品还有业务成果一顿夸，虚的比较多，偶尔却又在某个技术点上讲得很细，会场效果是只有在主管有提问的时候有人抬头看一眼。

从上面几个故事，我意识到其实问题的根源来自人根本的惰性：不爱思考。尤其是当遇到的场景是单方面接收消息的情况下。就好像师弟听师兄讲业务，下属听主管安排工作，参会人旁听式参会，本质上是信息接收者的脑子在 standby 状态，即：没动脑子。这是观众的错吗？固然有，但不全是

有一个现象：我们在开会时做分享现场写代码秀总结很多人不愿意看，但是却有大把人愿意看别人打游戏比赛的直播。究其本质，现场写代码，观众并不知道你的业务问题是什么，或者你的问题太宽泛跟我平时做的事情完全不着边，也不会对我的业务产生什么帮助，所以我也懒得动脑子想。而反观直播，观众知道这游戏是怎么打的，知道会如何发展（了解过），看到主播在玩的时候遇到的强烈攻势会认真观察主播如何化解（能带入），并且会想着甚至尝试会不会用得上这种技巧（能用到）。这不就是小学老师就常常说的这句经典：课前预习，课上提问，课后复习么 所以想了一个词：RPG式的沟通，核心是受众不再作为 NPC ，让受众成为主角带入到故事里。从世界观（问题背景）到任务（工作目标）都主动探索。也就是需要先提出一个让他能理解、能带入到自己的工作场景的、有趣的话题并且产生思考；沟通过程持续保证对方在思考，例如通过一些幽默的表达方式或者适当设问的表达方式刺激对方思考；最后如果这个沟通是有后续的实施的，需要随时跟进，最好能让其主动反讲，因为没法保证此次的沟通是完全 100% 信息传递的。

好累，还是觉得三体的沟通高效。
