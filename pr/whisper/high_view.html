# 职业 - 视界

如果给你一个机会让你现在就坐上你老板的位置，你觉得你能否胜任？差在哪儿？ 可能因为社会上太多德不配位的现象，导致很多人觉得有些位置放头猪都能飞起来，这种没什么讨论的价值，只说那种我觉得给我也坐不稳的 case 我们都知道，军事无非三个级别：战略 -> 战术 -> 战力，不管是部队，还是个人，要做成一件事的要素都是这三点，三者是不同程度的能力缩放，就像走路需要看地图，然后识路，然后凭脚力。我有个同事经常说一句话：不能用战术上的勤奋掩盖战略上的懒惰。我也补一句：也不要用战力上的苦大仇深来掩盖战术上的怠慢。意思是公司的市场决策 -> 部门的计划 -> 团队的综合实力；主管的业务思考 -> 业务的设计实施 -> 一线员工的能力；个人的年度规划 -> 业务的设计 -> 技术实力与投入，都符合这个 patten。时常有听到一些人在抱怨自己付出的多，得到的少，会想考试前几天熬夜多么辛苦，最终却没考好。

不想讲道理，还是想讲讲身边发生的事情。 这个季度换了一个新的主管，是一个社招的 p8 ，职衔是高级前端技术专家，然而他实际上并不是很懂前端技术，是一个在天语手机、用友等公司待过，带过产品、技术、市场、运营团队，而前端团队这是他第一次带。从他刚来的时候我就一直在想，他真的能胜任吗？我们团队的人才也不少，但是 p8 寥寥几个，就连我们的大主管都是去年刚晋升的 p9。然后就忍不住会多关注一下他的日常举动，今晚上有件小事很震惊我，他每天都在写写画画，和大多 隔壁部门不怎么受底下员工尊敬的 p8 “高级钉钉聊天技术专家”不一样的是，他聊的少，每天都会画部门组织关系、技术层次关系、业务层次关系。我们一直对 80 人的团队支撑了 30 多个业务线，数千个应用这件事非常的自豪，但是我来了两年多，至今都没有理清楚到底这些产品之间的关系是怎样的，只了解自己手里相关的十几个产品，而他来了不到 2 个月，就把这些东西从业务属性层面的关联分析得比我清楚的多了。 最近这几天休息一直都不太好，早上总是会在凌晨就醒过来，活着半夜睡不着，因为最近职级换了之后就总是会想，自己是否能承担得起来这个 title。对于这个新的职级，我从开始的两个关键字：业务能力、技术实力，增加了一个软实力。

关于软实力这件事，还是举几个例子讲。 一杆到底的解决问题能力. 说的是一种责任心，不过如果直接说责任心，似乎有点鸡汤了。最近我在负责一个底层容器技术，一个业务接入的时候出现了大量的 bug，其实这些基本上都跟我没有什么关系。但是大主管分了一个持续了半个多月很严重的 bug 让我解决。迫于主管要求，不得不抽时间解决，后来大概查了 30 分钟，基本定位清楚原因了，如果是以往，我可能就把原因和解决办法在群里一发就万事大吉。但有点“怕被骂”，而且负责这个业务的同事做事情效率不敢让人恭维，所以还是想把事情处理的完整点，就直接提交 pr 并且在群里 @ 了对应的几个人。然后就忙其他事情去了，这时候是下午 3 点左右。到了晚上 9 点，大主管在盘点问题的时候，问我这个问题的进展，我就 @ 那几个人，然后没一个人吭声，打电话问才知道他们都觉得自己不用管。我瞬间懵了，不用说，这就是我的失职了，于是紧急召集人，半个小时左右做测试和发布，事情总算告结。这事情给了大主管很不好的印象。总结起来，一杆到底的能力，就是让人放心把事情完全交给你，而不是相信你有办法解决。

逻辑清晰的业务魄力。说的是对业务的思考层次能力和主动能力，清晰的知道，因为遇到什么问题，所以我的问题梳理链路是 ABCD，然后我要分别解决 ABCD 问题，最后拿到 E 结果。在大公司开会，经常容易陷入一个奇怪的现象：所有人都在开会，但是所有人都在看自己的电脑自顾自，少数几个听的也只是光接受输入，没有辨识和转换，说难听点就是带了耳朵，没带脑子。最近在做一个新的项目迭代，是最整个产品的 2.0 升级，从产品概念到功能设计都是个换新，这是背景。但在项目评审中我发现，PD 在设定了一个产品基调大旗之后，实施的功能却天马行空，好像这个大旗就是个篮子，什么都能往里扔，典型的因为 A 所以要做 B ，因为“我觉得”可以拿到 C 结果，完全没有逻辑性。甚至某些功能完全是凭感觉来设计的，我少见的在会议室跟 PD 争论起来，不同意他的方案，但是到最后发现孤立无援，因为其他人仍然在低头看电脑，没有搞清楚我们的争执点在哪儿。而我自己讲到一半也说不下去了，因为我有个不太好的毛病是思维太跳跃，A 到 B 再到 C，我有时候想得太快，会忘了 B 是什么，只记得 AC ，而这地方成了我辩论点的弱势。往往只能会议后才想起来，但是会后又容易被不了了之。这里是有两个问题：一个是清晰的逻辑，一个是能说服业务方的业务魄力，不仅要能理直气壮，也要理不直气也壮，因为我明知道我现在不能被说服，我就不太可能有效执行。而这往往就直接导致了业务的是否顺利执行甚至是成败。

技术变现的能力。这几年我写了不少自己觉得很有意思的东西，比如若干个 chrome 插件，抽象语法树编译器，微应用桌面开发框架，等等，自己觉得技术含量还是蛮高的，但是并不是很能帮我得到精神快感以外的东西，说白了就是也没帮我提高绩效。后来有一次发现了这事的本质：我造的是工具，在工具没有发挥其最大生产力之前，是不会有什么业务价值的。我们的 p9 大主管今年开始给自己定了个目标，要成为一个“技术商人”，多少和我说的这个有点类似。变现有很多种，把技术变成商品，或者把技术变成商品的一部分。我们很多普通人，其实都只能做到后者，而如果能做到后者其实已经很不容易了。后来我做了个事情：去分享我的技术，写内网论坛文章，吸引更多开发者使用我的东西，去钉钉做技术分享，让人了解我的东西，去争取在业务中使用我的微应用框架，把技术和业务成果建立链接。也就是我上一篇文章讲的事情，这些事后来帮助我拿到了不少的成绩。

**大局的视野。**前端技术圈子，有个通病，就是喜欢造轮子。我经常有分析这个事情，结论无非是：造轮子门槛低，容易获得自我满足感。轮子好的，往往是上一点说的变现能力强，而事实上大部分都是无法变现的，今天写了明天自己都想骂自己的也不在少数。我想要说的大局技术视野，是指时空上的价值放大能力：有前瞻性，未来业务会有如何的走向，自己的这个轮子是业务这个南辕的北辙？还是领航员；团队范围接受性，是否能够打通现存不同业务与人员，还是另起烟囱污染环境。

最后还是精神力。很赞同某位坚持写朋友圈的大佬说的，早起一个小时和晚睡一个小时，完全是不同的感受。精神力往往能决定很多事情：记忆力、可持续战斗力、创造力、逻辑性。给自己个 Flag，每天争取 12 点睡，7点起吧，如果业务不那么忙的话。
