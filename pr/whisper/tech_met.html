# 所见 - 技术大会众生相

参加了一场技术分享大会，去之前其实就不太抱幻想有什么大的收货，毕竟太多讲师是冲着刷脸去的，跟 github 刷星也没什么本质上区别，甚至是 github 真的需要实实在在的代码，而技术分享只要个 ppt 就够了。 听完几场，总结起来有这几类

  • 晋升答辩型：谈团队内如何从粗糙的开发模式中发现问题，然后做了，内外部的技术分析，做了几次版本迭代，每次迭代的效果和下一步的计划。（总结展望也属于下一步计划、业务产出属于迭代的效果）
  • 产品发布型：粗浅介绍了自己的业务整体涉及思路和发展历史，对外有哪些产品 feature，各 feature 的涉及思路、特征、用途用法。给出一些业界同类产品的近况、 bench mark 、特性对比；
  • 代码博客型：介绍团队内遇到了什么问题，代码上如何解决的，会 show 部分核心代码设计思路。会提一些业界的框架库、做法，或者自己的实现方法如何落地
  • 漫谈技术型：大部分都是网上众所周知的事情，讲讲常见的热门技术其背后对应的发展历史、原理、缺点、数据，谈谈使用场景和展望，顺带会提一些自己（的团队）在做的事情在这个领域的什么位置
  • 思路分享型：结合业界的热门技术，给出一个如何实际落地到业务的方式的分享。目前看大部分技术大会很少有人会这么做，可能是这样看起来比较 low 不那么高大上？或者是这样看起来让有些受众觉得不那么“丰满”，不像晋升答辩型这类的看起来那么面面俱到看起来很「值钱」，又或者是有的人可能怕自己的核心代码被人抄了去所以不爱这么分���？但我个人比较喜欢这种，会有能学到的东西的感受

我自己是思路分享型，因为我自己就喜欢这样的，可以拿一些点子并且走通 hello world，让人获得一些启发、 其次我喜欢的是代码博客型，至少能看看别人在做什么，总结到了些什么实实在在我也许能用得上的东西。 最讨厌的是晋升答辩型，拿一个公众场合给自己走秀，吹嘘自己，浪费别人时间。
