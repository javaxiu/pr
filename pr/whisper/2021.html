# 职业 - 2021年终总结

这一年，自己的职业生涯发生的巨变，可以说是沧海桑田了。很多事情，难以列举，以一些总结受用至今的道理，串联一些感受良多的发生在自己身上或者身边的事情。

# 二八定律

关于团队价值 今年团队发生了一次重大变化，团队高层从副总裁层面开始调整，一大票能力强、业绩效果突出的人划到钉钉。事情突如其来，却又顺理成章。原因是副总裁不穷在前年的健康码项目中功勋卓著，而钉钉在产品高峰期面临商业转型，需要不穷这样能将钉钉带出社会影响力。而健康码背后依托的宜搭则成为了集团瞩目的明星产品。宜搭从我入职开始，就一直不温不火，我一度以为是一个烂尾楼项目，平时也极少关注其发展，更多是在使用宜搭“内部专业版”乐高。事实上，宜搭和乐高也一度面临生死关头。而很久之后我才知道，它的创始人、我的前老板，从十多年前就一心在打造这样一个产品，这个产品如果没有我的前老板扛着，可能今天就不存在了。 这件事给我一个很大的启发，在无人看到处，以未来的视角，坚定一个正确的事情，十年磨一剑，没有这样的魄力是难以完成自己理想的事业的。 一个团队，固然有公司赋予它的使命，就如我原团队，前端需要支撑整个集团内部办公产品的研发，都是DAU 数十万的产品，压力可想而知。因为这就是公司花钱雇佣我们的目的。 可问题在于，即便是这种压力下，前端团队仍是横向甚至接近外包性质，从来都是与业务部门同工不同酬，在完成团队使命之外，如何创造自己独一无二的价值一直都是整个前端团队、每个前端开发需要思考的问题。我前老板给出的答案是创造自己的产品，从一开始的 UIPaaS，一路演进到现在的宜搭。事实上，宜搭就成为了这个团队产出的惊喜价值。 而这就是我想说的「团队价值 - 二八定律」。被赋予的使命是那「八」，是当前生存的必须做的事情。而创新则是「二」是为了让未来能活得更好的事情。结合自身业务的特殊性，以自己从业多年的经验来创造这个「二」，来为自己未来的生存铺垫出来一个无可替代的价值。「这件事为什么应该我来做」，「为什么是现在做」，「和别人有什么不同」这三个问题就都能得到一个很好的解答。 团队的「八」可以是指 80% 的时间，也可以是 80% 的人，或者 80% 的资源。 当然，这个理论对于个人也同样生效。这并不是说需要有 20% 的保留为私己，而是说在 100% 为团队基本价值之外，要有对等的 1/4 投入是在培养自身的竞争力。来自高层分配的任务，没有单独指出的是，希望在投入这些人力物力之外，最好能创造出来一些意外惊喜，这些惊喜就是这 20%。因为高层一旦提出来想要的事情，那便属于是 80% 了，否则如何叫做“惊喜”

关于团队管理 因为阿里崇尚的是每个人需要有自己的独立思考，自己在每一个财年，是需要有自己给自己写财年规划的，这和很多自上而下的分任务不同。初入阿里直至很长一段时间，我都每太理解为什么需要写，甚至很多次我都是把年度规划写成「把老板的一句话目标，用一堆说了跟没说什么一样的废话详细分析分析」。相信这种做法的在团队目标大锅饭的团队里，并不在少数。 也是从今年开始，我第一次带了一个小规模团队，对于这件事我才有了改观。之前写了一篇文章，讲如何有效沟通 (opens new window)，就有一段故事，我在对一件事情想的很清楚了，从产品目标到技术分析，再到详细编码策略，就只是没有时间去投入开发，于是把这件事分给其他人，最后发现完全是南辕北辙。这件事给我对于团队管理自信带来一个很大的挫败感。我自问并没有哪里没说，也没有说错什么，过程中也有反复 check，可最后为什么会如此。 后来我 book 了一个老板的茶歇时间，他从业二十多年了，也有相当多的社区流行开源产品。我就找他说起这件事，得到了一些启发。我就慢慢意识到一个问题，人都是有独立思想的，需要的不是完整的工作流程输入，而是目标指引。否则要么是逆反心理、要么是信息接受失真、要么是主观观念偏差、要么是背景了解度不足。我需要做的是给予一个完整且正确的目标，然后其他的作为闲聊建议，并时刻盯紧目标的误差是否在可接受范围之内。这就是一个团队管理需要坚守的「二」。剩下的为了完成这件事需要做的 「八」应该由每个人主动创造和发挥。对于团队而言，如果他有自己再分解再创造出来的有价值的「二」，那是最好的；如果没有，也应当以指路人的身份给一些建议帮助其发挥自己的创造性，并为他自身带来团队生存自信心。 再后来我听了一些课，才知道这个「二」是指马斯洛生存动机里的「生存自尊心」。即: 没有任何一个团队成员愿意以机械执行者的身份，做一些随时可以被取代的事情，那样只会让人逐步失去生存价值，成为可以被热替换的干电池，甚至连零件都算不上。因为零件下岗还能再就业，干电池耗费自己的青春岁月换来废弃有毒标签。任何把人当做火车轮换的轮胎的团队，最终都无法留住有创造性的人才。

关于架构设计 最近发现，一些工作稍微多些年的人，对于自己擅长的技术有一种魔怔的偏执，对于别人想做的事情又有一种近乎鄙视的不屑。总结一下，大概是「文人相轻」，也就是对于自己的经验过分自信。我有尝试在新团队做一些新的事情，面临的支持远少于反对，每次提出来的 A 方案，总有 BC 说为什么不用 DE，并且坚称 DE 更佳。 对于一件没有讨论标准的事情发表优劣之言，并且持续条件性反射的反对，我们可以简称为抬杠。由于变更团队带来的这种差异，我为此不适应、苦恼了一段时间之后，渐渐想明白一个道理，我要完成的事情不是分高低，我需要对目标坚定，比如研发效能提升，比如团队积累，比如对产品研发得到一些预判带来的技术侧路线规划。这才是我需要坚持的那个 「二」，在这个框架之下，任何想要的变更，都在允许范围内的「八」。所以我在团队里的风格稍作了一些改变，事先问他们的喜好，对于他们提出来的杠我直接无条件接受，毕竟抬杠的都只是局部技术，不影响全局。而全局问题，我只需要跟主管层面达成共识即可。而兼容并包的「八」最后还能落下个好名声 -- 虚心接受团队同学的建议。 好的架构设计（不特指程序，只要是一件团队协作的事情），应该允许有「八」的随性发挥，这也是上文所说的「生存自尊心」，如果任何事物都给人规定条条框框，那么团队也好，产品也好，是无法达到最佳目的的。区别在于有的人心里反对，有的人嘴上抬杠，有的人行动反对，有的人兼备。

# 内卷

这个话题今年有很多文章都在聊，获得喝彩的大部分都是煽动企业与员工对立的文章。我倒想另外一个问题，「内卷」的反义词是什么？ 先解释什么是内卷：比如经典的内卷的释义，电影院里最前面有个人站起来，后面的人不站起来就没法看到屏幕；比如公司里一个员工开始加班，其他人不加班就得不到好的绩效，于是所有人加班，但是第二天又起不来，最终变成所有人下午才上班，晚上干到凌晨，然后企业微信就成了起夜微信。基于这种现象，努力工作的、多掌握几门技术的、把事情做得更极致的人，一律被贴上标签「卷x」，并且群起而攻之。 那为什么电影院里会有第一个站起来的人？其实可能只是ta想要看得更清楚而已。第一个加班的人，可能就是他当前工作效率不高急于赶完当天的任务。而「站起来」这个反应其实可能是无奈之举。如果电影院能做更好的结构设计让第一排看得清楚，如果第一个加班的人认识到这个加班无意义，需要在这次加班里找出问题症结，然后在接下来的工作中总结出更高效的工作方式，那接下来不就能 “大庇天下寒士俱欢颜” 了么？ 对于困境不是以一味的压榨时间和重蹈覆辙，突破出来一些新的途径，我们可以称之为「创新」，这就是内卷的反义词。 一项工作重要的并非它周而复始持续不断的走泥泞的土路，而是这项工作阶段性完结的一个总结。要判断一个老板是不是一个好老板，也是看他是想听你这个工作是不是按时完成了延期了甚至瞪你两眼并提醒你下次注意，还是想听你遇到了什么坑怎么解决的是否能为团队提供普适性的提效谏言。遇到前者，赶紧跑为上策，因为他给绩效的方式就是谁卷得最秃头的就绩效最高；如果遇到后者就是万幸千万珍惜。 人的历史就是在 【一种生存方式 -> 此方式的极致性能 -> 探索新的方式 -> 新方式的极致性能】 这样一个不断轮转的周期里循环往复拾级而上的。而探索创新，自然是在当前阶段性能遇到了瓶颈预兆时采用上述的「二八定律」去为未来投资，从而在旧时代的平台期突破到下一个阶段。就好似《三体》里的材料物理到量子物理的跃迁，不同的是三体里有智子的技术遏制，而现实中没有，随时可以创新，随时都可以突破。人们可以在 5G 还未普及时就投入研发 6G、甚至畅想 7G。人类历史成功的企业，大抵如此。好比华为研制 5G 时，别人可能还在为那 3G 的点点流量斤斤计较。 内卷不停，然后后知后觉被突破的人、团队、产品，面临的就是被别人用下个阶段的实力对你降维打击。

# 领域

阿里之所以伟大，我看到的一点优势就是上述的从制度层面帮助每个人发展主观能力。无论团队的大小，都应该发挥 1+1 > 2 的实力。因为每个人都在为自己「二八」里面的「二」建言献策。从 TL 层面提出自己团队的创新核心竞争力，到每个人为这个核心竞争力贡献自己的一个个「二」，让每个团队在完成既定目标的同时增强下一阶段的实力。但是每个人如果遇到了每件新的事情，都眼高手低，设计了很多业务、团队所不需要的事情，那岂不是浪费人力物力留下一堆鸡毛？ 对于个体（团队或者人）而言，每个创新其实都是在塑造自己的一个特有能力集合，都是在往自己的这个名为「领域」的篮子里放鸡蛋。 对团队而言，需要何种领域的人才、每个下属的能力指数偏向，其实管理者是一目了然的。比如一个网页产品，关注的是 DAU、产品转化率、多端覆盖率，那所需的人才特质就了然纸上：稳定性能监控、流量转化分析、跨端技术、特定端技术。结合已有的人才特质，为每个人量身打造一个领域特征。比如有人专业负责访问速度稳定，有的人负责多端协同研发技术，有的人负责移动体验，有的人负责流量跟踪分析技术，然后分工学习业界先进技术，而不是每场战斗都一窝蜂人海战术。要知道，历史上我国虽然靠人海战术战胜过很多强国，但那都是以人口锐减为代价的。反应在团队里，就是每个个体学无所长，精无所精，最后每个事情都是差不多完成，然后个体价值都无法实现，人员流失，产品停摆。 需要时常自问：我(我或者我的团队) 的特质是什么，如果我今天做的这个产品因为某种原因停下来了，凭什么让公司继续雇佣我，我有没有留下可以再次使用的价值，比如经验、抽象产品。我就经常有这种焦虑，我称之为「核心竞争力焦虑」。在阿里，我的领域是「PC跨端技术」，标签很鲜明，自认为在国内几家巨头公司应该是没有比我了解更深刻的了。不过这不是什么值得夸耀的事情，而是每个人赖以生存可以理直气壮告诉别人我擅长的事情，如果没有做到，我会混不下去。 然而这种底气，在我换了工作之后崩溃消失殆尽了，自信心被完全挫败。因为新团队不需要这个技术。 于是我开始重新审视自己，我还有什么。我喜欢看技术原理，我会把很多开源的��术从设计思想和原理上弄清楚其所以然。即便是「PC跨端技术」，我也是通过看了不少源码和逻辑去掌握的。在新的团队，他们在用某一项我听过但是没用过的技术，于是我决定把源码全部读完，甚至后来还提交了 pr 并且顺利 merge 了。随后在工作中也开始推动一些基础设施的事情。于是我的领域特质又慢慢形成了：「移动端跨端基础技术」。虽然还没成型，姑且先这么叫，用来将后续我的各种小的「二」往这里靠近，组织起来我的日常工作。

# 团队与个人

我近两年发现自己身上不知道什么时候多了个特质：能从不得不做的工作中找到兴奋点，这个技能让我多次获益。比较有代表性的一件事情，是去年在前团队发生的。 当时产品经理突发奇想，给我提了一个新的需求，以我当时的话语权还有价值判断，这件事只要我说我不做，就可以轻松甩掉，也不会得罪人，而且我的主管也必然会支持我。但是我从产品经理的角度推想了原因，这件事对他而言也许是他业务创新里的那个 “二”，做了这件事能对他的 kpi 很有好处，并且接下来对于主线业务的推进、团队凝聚力都有提升，就硬着头皮答应了下来。可面临的问题是，这个事情挺无聊，真的是纯业务，给��包也一样做，做不出什么新鲜玩意儿，怎么办？于是我尝试分析这个事情可能面临的技术点，大概有3种实现方式，然后我发现其中一种方式能够为我们团队的主线业务技术栈补上一块拼图，在未来的业务里面说不定还可以发挥出来一些奇效，只是这种方案可能需要额外多投入一些时间。心里就算了一笔账，如果我用一般的方式应付，相当于花10块钱打了个水漂，但是我如果花15块钱，我打完这波水漂之后还能赚点额外奖品，那毫无疑问是后者。 于是，用周末和晚上加班的方式，额外用有附加产物的方案帮他做成了这件事，并且在团队里分享了该技术，皆大欢喜。 今年换工作前，其实想了很多想要做的事情，想着现在这个年纪，想要和年轻能熬得起夜的人相比，能有一些优势，并据以为资本，就根据自己的领域标签，以技术栈有差距为由，明确拒绝了好几份待遇相对优厚得多的 offer 。然而事与愿违，挑选半天得罪了不少公司的 hr 之后，选中了现在的工作，结果却并不如预期那般合适我。于是，和很多加入新团队的信任一样面临了一样的问题：工作的事情我不喜欢，怎么办？ 就想到了上面的这个事情。当遇到一个事情，需要先权衡是否做，如果是那件不得不做的事情，就判断能否做出来15块钱的效果，进而从其中获得可沉淀价值的方式。眼前的事情，也是如此，来都来了，得创造点个人价值和团队价值再说，否则就像个逃兵不是。一定要做些事情出来，让事情因我而不同

# 手留余香

以前有听一个课程，讲个人财富管理，每个月需要分别给自己的长期学习、人脉关系、家庭关系、金融投资分别划分一笔固定的钱，用来让自己各方面持续发展。可惜的是这方面的规划，今年也还是没做好，习惯性攒钱省钱，不舍得花。不过除了钱之外，自己的时间精力投入，倒是有些变化，有些以前坚持拒绝不肯做，觉得从业务价值角度没什么意义的事情就不肯做、硬刚到底。但是今年会去考虑可能跟产品经理的个人 kpi 有关，可能跟设计师的审美亮点有关，设身处地去想，换做我，如果别人答应帮我做一件我没有底气让别人帮助我的事情，那是件值得感激很久的事情，对ta 整个人的印象都会有很大的改观。 当我用个人的知识、经验，去补位帮一些
