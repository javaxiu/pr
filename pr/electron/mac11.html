# Mac11.4权限问题

# 问题现象

# 现象分析

# TCC

通过解锁 mac sip 权限之后，读取 sqlite3 /Library/Application\ Support/com.apple.TCC/TCC.d 的列表看到 db ⾥的数据如下，正常能够显示图标的⼀⾏是 client 显示的是 com.alibaba.CloudUEM 这 样的 bundle id，⽽异常的⼀⾏是直接指向了 app 的⼊⼝

# 权限申请机制

CloudUEM ⾥，前端需要申请投屏权限的流程是前端给 native 独⽴进程发⼀个消息，然后由 native 来唤 起投屏进程的，⽽判断是否有权限是 native 通过直接拿⼀个抓屏流，如果能获取到⾮空的流则认为是有权 限 native 端判断是否有权限

# 测试唤起其他投屏进程

因为此问题在阿⾥郎⾥并不存在，只在 CloudUEM ⾥出现了，两者的区别是阿⾥郎是之前直接调起投屏进 程，⽽ CloudUEM 是通过 native 来调起的投屏，因此这个问题很明显是在 native 端的调⽤产⽣了问 题。尝试由前端调⽤其他投屏的进程看是否存在此问题

# 唤起钉钉的截图助⼿

钉钉的安装包⾥⾯有⼀个截图助⼿ /Applications/DingTalk.app/Contents/Frameworks/ScreenShotTool.app 尝试着由 CloudUEM 直 接唤起，发现权限申请关系是正常的，归属到⼀个 Bundle Id 上，相应的弹出界⾯也是正确的

# 唤起⾃⼰写的 Swift Mac 应⽤

⾃⼰⽤ xcode 写了⼀个简单的 swift 应⽤，原理是直接通过 process 调⽤⼀个录屏。表现和唤起钉钉⼀ 样是正常的，⼀样的显示 CloudUEM 的 bundle id

# 钉钉的做法

钉钉有截图+会议共享屏幕两个功能都是使⽤到录屏权限的，但是却是分开的两个权限申请，仅授权⼀个的 情况下另⼀个会⽆法正常获取到权限。但是从效果上看，⽤户并不会觉得有什么异常，我们 CloudUEM 遇 到的问题也仅仅是因为图标显示了 CloudUEM ，⽽⽤户安装的 app 名称是 「联呗」，会让⽤户看到这个 会有意外的感觉。

# 根本原因

经过两三个周末逛外⽹ Mac app 开发的论坛，看到有⼈也遇到了类似的问题，看完他们的讨论发现 了这样⼀个零⽇漏洞的分析 https://www.jamf.com/blog/zero-day-tcc-bypass-discovered-inxcsset- malware/ 简要概括⼀下如下，就是攻击者可以通过遍历⽤户电脑上有某权限（举例录屏权限）， 然后将该 app ⼊⼝替换为⽊⻢⼊⼝，由⽊⻢⼊⼝再唤起宿主⼊⼝，给⽤户看到的效果是 app 是正常启动 了，并不知道后台也启动了⽊⻢，⽽⽊⻢本身获得了和此 app ⼀样的权限，不需要⽤户再次确认授权 为了解决此漏洞，Mac 11.4 合⼊了⼀个 patch ，原理是每次运⾏都会检测可执⾏⼊⼝的合法性 “detect and prevent anytime this vulnerability is potentially being abused.” ⽽检测的机制是检查 可执⾏⼊⼝的数字签名的有效性。 对⽐ CloudUEM，native 进程和投屏进程确实没有签名过，是通过 zip 包塞到 .app 整包⾥，绕过了 签名，同时在⽤户⾸次运⾏时做解压处理，因此导致了授权链路的异常。因此只需要把 zip 去掉，对所有 内容做签名就可以解决问题。 ⽽之前 CloudUEM 为什么要把 native 可执⾏⽂件打包，原因是 native ⽂件较多，当时遇到了 app 签名⼀直失败，和当时的 Mac 开发赵睿同学⼀起尝试了各种办法查了各种⽂档，⼀个多星期都没有解决， 最后商量就⽤这种把 native 进程全部打包掉运⾏时再解压的办法。⽽现在⼜不得不⾯对这样的问题⽽且只 能⾃⼰⼀个⼈查了，艰难的排查过程略过，当然这次也是花了好多天时间，结论是被签名的 app 程序包⾥ 不能包含 ._ 开头的⽂件、.DS_Store、__MACOSX 这些垃圾⽂件，否则签名时虽然会忽略且正确签名完 成，但是在⽤户电脑上执⾏的时候会报如下错误

# 总结

由于 Mac11.4 的安全机制会⼀直存在下去，以上分析会在后续的 app 开发都需要注意的：

  • 如果业务有不得不动态下发的可执⾏程序，例如动态模块，UEM 以后也许也会遇到这种安装时没有，

但是根据租户可能需要动态加载可执⾏程序的，那么这种场景合适以独⽴进程的⽅式唤起，⽤户直接 给投屏进程授权，⽽不是通过主程序申请权限。具体做法是 open /xxx.app --args 参数 这样的⽅式

  • 如果下发的就是整包，不会动态下载则要注意程序内所有⽂件都需要构建时签名。签名需要注意 ._ 等
