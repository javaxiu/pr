# 边下边播 - Electron 嵌入播放器？

迅雷支持边下边播放技术，这一点让我实现我也不知道要怎么做比较合适，毕竟视频流是没有下载完成的，编解码肯定是用 native 实现会更顺畅，但是如果非要用 electron，我想到的方式可能是这样的：

  1. C++ 实现下载和视频数据解码
  2. 点击播放时，通过 ffi / rust 之类的方式，调用 C++ 的解码流，获取到 stream
  3. 前端创建 video，然后将上一步拿到的 stream 作为 srcObject 传给 video播放。这一步也可以用 canvas 实现，道理是差不多的，顺带一提，飞书的视频会议似乎就是这样实现的，native 将视频数据流采集到然后传递给前端的 canvas 绘制

那么迅雷是怎么实现的呢？ 通过 Element 检查，发现该区域只是一个占位的 dom，实际上并不负责播放工作，只负责立即播放按钮的展现和触发播放器。

然后通过源码查找相似的位置，找到了播放函数，然后打断点验证该函数确实是播放位置：
